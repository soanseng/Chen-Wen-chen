# 遊戲流程大綱

> 玩家扮演一位現代調查者，在促轉會解密檔案公開後，透過一套樸素的檔案瀏覽系統，逐步拼湊真相。
>
> 介面設計：類似終端機或舊式文件管理器，黑底綠字或米色紙張質感。輔以簡單像素風格圖像（印章、紙張質感、文件圖示）增添年代感。以文字為主。手機優先、電腦也能玩。

---

## 敘事框架

**時間設定**：當代（202X年）。一批解密檔案剛被公開，玩家是接手整理檔案的研究員。

**開場**：螢幕顯示一段簡短文字——

> 民國70年7月2日，一位旅美學人被帶離家門。
> 隔日，他的屍體出現在大學校園裡。
> 四十年來，真相始終未明。
> 這些是剛解密的檔案。請開始閱讀。

**核心循環**：
1. 閱讀文件
2. 標記可疑段落（長按/點擊）
3. 系統偵測矛盾連結
4. 觸發連結 → 解鎖新文件 / 新推論 / 推理簿欄位
5. 重複直到所有可解矛盾被觸發

---

## 章節結構

### 序章：歸鄉（情緒基調：溫暖、帶有不安）

**初始文件**（玩家一開始可閱讀的）：
- 📄 陳素貞家書（DOC-EXT-04）——妻子寄回臺灣的信，提到丈夫可能回國任教。日常、溫暖。
- 📄 出境申請表（DOC-OFF-01 節錄）——在機場申辦出境，簡單的行政表格。
- 📄 新聞剪報：70年7月4日《聯合報》——「台大發生離奇命案」。

**玩家行動**：
- 閱讀家書 → 感受日常生活的溫度
- 閱讀新聞剪報 → 產生時間落差的衝擊（信中計畫返美，新聞報導死亡）
- 標記家書中「返臺任教」 + 新聞中「歸國學人陳屍校園」 → 觸發矛盾

**解鎖條件**：標記家書與新聞的時間落差 → 解鎖第一章

**情緒節奏**：
```
溫暖的家書 ──▶ 出境申請的平凡 ──▶ 離奇命案的標題 ──▶ 不安
```

---

### 第一章：監視之網（情緒基調：壓迫感逐漸升高）

**新解鎖文件**：
- 📄 彩虹資料第0487號（DOC-SUR-01.1）——最早的監聽紀錄，一通給施明德的電話
- 📄 信件攔查紀錄（DOC-SUR-02）——家書被拆開檢查
- 📄 校園安定小組登記表（DOC-SUR-04）——大學人事室主任的另一個身份
- 📄 報告表（DOC-SUR-03）——警總對陳文成行蹤「瞭若指掌」的證據
- 📄 特種資料：電話監聽（DOC-SUR-01.5）——康明昌被稱為「陰謀份子」

**可觸發矛盾**：
- **C-01 監控範圍**：標記家書被攔截 + 校園安定小組 → 解鎖更多監控文件
- **C-03 境管局共謀**：標記出境限制 + 報告表中「假藉名義」→ 解鎖第一次約談紀錄

**解鎖條件**：觸發至少2個矛盾 → 解鎖第二章

**特殊設計**：
- 每解鎖一份監控檔案，檔案瀏覽器的背景會稍微「暗」一點（可透過 PixiJS 覆蓋層或 CSS 漸變實現）
- 彩虹資料的「極機密」紅色戳記是視覺焦點（以 PixiJS 像素風 sprite 渲染）
- 玩家意識到：連最私密的家書都被國家機器翻閱

**情緒節奏**：
```
一通電話 ──▶ 一封家書 ──▶ 一張登記表 ──▶ 一份完整的監控報告 ──▶ 窒息
```

---

### 第二章：約談（情緒基調：緊張、封閉、無處可逃）

**新解鎖文件**：
- 📄 第一次約談紀錄（DOC-INT-04）——6月30日，公關室，「先軟後硬」
- 📄 調查筆錄（DOC-INT-01）——7月2日正式約談
- 📄 錄音譯文節錄（DOC-INT-02）——殘缺的對話
- 📄 陳素貞偵訊筆錄（DOC-WIT-04.陳素貞）——穿著細節
- 📄 鄒小韓答覆表（DOC-WIT-05）——12項問答

**可觸發矛盾**：
- **C-04 約談目的**：標記錄音譯文中暗示合作的段落 + 官方「澄清疑點」說法
- **C-06 是否經同意**：標記警總「徵得同意」+ 陳素貞「毫無預備」+ 甯漢彤「未事先照會」
- **C-02 錄音帶遺失**：標記監察院「聆聽六小時錄音」+ 國安局「刪除敏感部分」簽文

**解鎖條件**：觸發C-02（錄音帶遺失）→ 解鎖第三章
- 解鎖時螢幕顯示：「你正在尋找的錄音帶，在這份簽文之後，再也沒有被任何人聽過。」

**特殊設計**：
- 錄音譯文以特殊格式呈現——文字逐漸出現，模擬錄音播放（React 元件定時渲染或 PixiJS 文字動畫）
- 「〔略〕」標記處留白，暗示被刪節的內容
- 約談對話中，訊問者的話用粗體，陳文成的回答用細體
- 最後一段對話後，螢幕慢慢變暗——「約談至此結束。此後陳文成再也沒有回家。」（PixiJS 覆蓋層淡入）

**情緒節奏**：
```
假境管局之名 ──▶ 突襲帶人 ──▶ 審問的壓迫 ──▶ 錄音帶消失 ──▶ 沉默
```

---

### 第三章：疑點（情緒基調：憤怒、困惑、拼湊的挫折）

**新解鎖文件**：
- 📄 法醫鑑定書（DOC-FOR-01）——傷勢細節
- 📄 現場勘查紀錄（DOC-FOR-05）——屍體姿態與衣著
- 📄 鄧維祥證詞（DOC-WIT-01）——聲稱填補6小時空白
- 📄 五名目擊證人筆錄（DOC-WIT-02）——聲稱目擊送返
- 📄 倪肇強證詞（DOC-WIT-03）——聲稱凌晨目擊
- 📄 魏契鑑定意見（DOC-FOR-02）——美國法醫的結論
- 📄 李俊億鑑定報告（DOC-FOR-03）——背部傷痕分析

**可觸發矛盾**：
- **C-01 送返時間 vs 死亡時間**：核心矛盾
- **C-05 鄧維祥證詞 vs 物證**
- **C-07 衣著矛盾**
- **C-08 陳屍處非第一現場**
- **C-10 結論先於科學證據**
- **C-12 護送人員身高**
- **C-13 倪肇強證詞崩潰**

**解鎖條件**：觸發 C-08（陳屍處非第一現場）→ 解鎖第四章

**特殊設計**：
- 文件數量驟增——模擬真實調查的資訊過載感
- 推理簿在此章大量展開，但每填一格就會發現新的空白
- 玩家可以在推理簿中嘗試連線，但某些連線永遠無法完成

**情緒節奏**：
```
法醫報告的冷靜 ──▶ 證詞的矛盾 ──▶ 物證的沉默控訴 ──▶ 「非第一現場」的震撼
```

---

### 第四章：掩蓋（情緒基調：系統性的恐怖、冷酷的官僚語言）

**新解鎖文件**：
- 📄 落體實驗報告（DOC-FOR-04）——排除自殺的秘密實驗
- 📄 汪敬煦呈蔣經國簽呈（DOC-OFF-05）——「已報總統」
- 📄 國安局「不公布」決定——「政治性大於法律性」
- 📄 偵查報告初稿與定稿比較（DOC-OFF-04）
- 📄 專案會議紀錄（DOC-OFF-02）——「全面監控家屬」
- 📄 家屬監控檔案（DOC-SUR-05）——案發後的持續監控
- 📄 國安局撥發工作費（DOC-OFF-06）——匪間諜案預算
- 📄 查禁書籍紀錄（DOC-OFF-07）
- 📄 美聯社記者事件（DOC-EXT-03）
- 📄 外交部投書運動（DOC-EXT-05）

**可觸發矛盾**：
- **C-09 落體實驗 vs 對外說法**——核心矛盾
- **C-11 報告初稿被修改**
- **C-14 嫌疑人主導辦案**
- **C-15 國安局「沒有檔案」**
- **C-16 汪敬煦升遷**

**解鎖條件**：觸發 C-09（落體實驗矛盾）+ C-15（國安局謊言）→ 解鎖終章

**特殊設計**：
- 此章的檔案不再是「調查真相」，而是「掩蓋真相」的紀錄
- 玩家第一次以加害者的內部視角閱讀——官僚語言的冷漠比暴力更令人不安
- 「政治性大於法律性」這句話以特殊格式突出顯示
- 家屬監控檔案中，陳庭茂被稱為「陳某」——去人化的語言

**情緒節奏**：
```
秘密實驗 ──▶ 「已報總統」──▶ 「不公布」──▶ 監控家屬 ──▶ 查禁書籍 ──▶ 冷酷
```

---

### 終章：未竟（情緒基調：無力但堅定——真相應該被追求）

**新解鎖文件**：
- 📄 歷次調查結論演變（DOC-OFF-03）——40年的變化並排呈現
- 📄 促轉會最終結論摘要
- 📄 美國眾議院聽證會紀錄（DOC-EXT-01）
- 📄 狄格魯《台灣之行》報告（DOC-EXT-02）

**推理簿結算**：
- 系統自動統計玩家已觸發的矛盾數量
- 展示推理簿——已填寫的格子與永遠空白的格子
- 每個空白格子旁有一行小字說明為什麼無法填滿

**結局呈現**：

推理簿展開後，螢幕緩慢滾動顯示以下文字：

> **你確認了以下事實：**
> - 陳文成在被警總帶走後，再也沒有回家。
> - 陳屍處並非第一現場。
> - 他殺之可能性顯高於自殺或意外。
> - 政府自己的實驗早已排除自殺，但選擇隱瞞。
> - 四十年間，錄音帶消失了、檔案「不存在」了、證人移居海外了。
>
> **但以下問題仍然無解：**
> - 那天晚上究竟發生了什麼？
> - 是誰？為什麼？
> - 完整的錄音帶裡記錄了什麼？
>
> 推理簿上有幾格永遠填不滿。
> 不是因為你不夠努力，而是因為有人花了四十年確保它們被填不滿。
>
> 然而，每一份被閱讀的檔案，都是對遺忘的抵抗。

**最後畫面**：回到檔案瀏覽器首頁，但所有文件都已解鎖——玩家可以自由回顧任何文件。

## 解鎖流程總覽

```
序章（3份文件）
  │ 標記時間落差
  ▼
第一章（5份文件）
  │ 觸發2個矛盾
  ▼
第二章（5份文件）
  │ 觸發 C-02 錄音帶遺失
  ▼
第三章（7份文件）
  │ 觸發 C-08 非第一現場
  ▼
第四章（10份文件）
  │ 觸發 C-09 + C-15
  ▼
終章（4份文件 + 推理簿結算）
```

**總文件數**：32份（+ 推理簿）
**可觸發矛盾**：16個（其中2個永遠無解）
**預估遊玩時間**：60-90分鐘（純文字閱讀）

## 推理簿欄位設計

推理簿是一張表格，玩家隨遊戲進行逐步填寫。部分欄位有選項，部分是自動填入：

| 欄位 | 解鎖時機 | 填寫方式 |
|------|---------|---------|
| 被帶走的方式 | 第二章 | 選項：自願赴約 / 突襲帶人 / 強制押解 |
| 約談真實目的 | 第二章 | 選項：澄清疑點 / 招募合作 / 施壓恐嚇 |
| 7月2日晚間行蹤 | 第三章 | 🔒 永遠空白 |
| 衣著異常原因 | 第三章 | 選項：自己更衣 / 他人整理 / 搬運所致 |
| 陳屍處是否第一現場 | 第三章 | 自動填入：否 |
| 死亡方式 | 第三章 | 選項：自殺 / 意外 / 他殺 |
| 鄧維祥的角色 | 第三章 | 選項：誠實證人 / 被利用的友人 / 合作者 |
| 完整錄音帶內容 | 第二章 | 🔒 永遠空白 |
| 偵查方向由誰主導 | 第四章 | 選項：檢察官 / 警總 / 國安局 |
| 掩蓋真相的決策鏈 | 第四章 | 自動填入：警總→國安局→總統府 |
| 是誰下令？ | 終章 | 🔒 永遠空白 |
| 真正的死亡地點 | 終章 | 🔒 永遠空白 |

## 技術考量

- **框架**：React（Vite 建置）+ TailwindCSS 樣式 + PixiJS 簡單 2D 像素風視覺層
- **存檔**：localStorage，自動存檔（透過 React custom hook `useSaveGame` 封裝）
- **文件系統**：每份文件是結構化資料物件（含 id、title、content、unlocked、read 等欄位），以 ES module 匯入
- **標記系統**：React 元件搭配段落層級的點擊/長按事件處理，系統比對已標記的段落對，觸發矛盾連結
- **推理簿**：獨立 React 路由或視圖，隨解鎖進度展開欄位
- **響應式設計**：TailwindCSS 響應式工具（`sm:` / `md:` / `lg:`），手機優先。手機用全螢幕文件閱讀 + 底部 tab 切換；桌面用左右分欄
- **像素風視覺層**：PixiJS canvas 用於紅色「極機密」戳記、紙張老化質感、文件分類圖示、章節轉場效果。裝飾性質，非互動遊戲元素
- **無後端**：純前端，Vite 打包為靜態資產
- **調性控制**：字體用宋體（Noto Serif TC），配色克制（黑、白、灰、少量紅色用於機密戳記）。無背景音樂。動畫克制且服務敘事——讓文字本身說話
