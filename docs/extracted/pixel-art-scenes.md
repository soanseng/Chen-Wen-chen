# Pixel Art 場景規格

> 本文件列出所有建議的 pixel art 場景，供 PixiJS 開發階段參考。
> 基礎解析度：320×180 像素，scale up 至視窗寬度。
> 風格參考：Papers, Please（色調、人物比例、簡約線條）。

---

## 核心場景（7 個，必做）

### 1. campus-life — CMU 校園授課

| 項目 | 內容 |
|------|------|
| 章節 | 第一章：學者之路 |
| 情緒氛圍 | 溫暖、平靜、學術的金色光線 |
| 互動方式 | 捲動顯示，進入 viewport 時淡入 |

**場景描述：**
Carnegie Mellon 校園一角。陳文成站在黑板前，黑板上有統計公式。窗外透入金色光線。書桌上散落論文與咖啡杯。

**需要的元素：**
- 人物：陳文成（西裝、眼鏡，28 歲學者形象）
- 場景：教室 / 研究室、黑板（含公式）、書桌、窗戶
- 光線：右側窗戶透入的暖金色光
- 動畫：粉筆書寫（微動畫循環）、窗外光線微微閃動

**參考風格：**
- 暖色調為主（金色、米色、深棕）
- 人物比例約 24×32 像素
- 教室採等距視角或側面視角

**技術考量：**
- sprite sheet: 陳文成站立 + 書寫兩幀
- 背景靜態，前景光線用 alpha 動畫

### 2. surveillance-web — 跨太平洋監控網

| 項目 | 內容 |
|------|------|
| 章節 | 第二章：海外的聲音 |
| 情緒氛圍 | 不安、暗潮湧動，暖色漸冷 |
| 互動方式 | 捲動推進，元素逐層浮現 |

**場景描述：**
畫面分為左（匹茲堡）與右（台北）。中間以太平洋海面連接。隨捲動，監控元素逐一浮現：電話線連接兩地、郵件被攔截標記、檔案從暗處浮出。

**需要的元素：**
- 場景左側：匹茲堡城市剪影 + 電話亭 / 書房
- 場景右側：台北（警備總部意象）+ 監控室
- 連接元素：跨洋電話線（五條，對應彩虹資料五筆紀錄）
- 浮現元素：信件（被紅色「攔截」標記）、檔案夾、監控攝影機圖示
- 動畫：電話線脈動、信件飛越太平洋被攔下

**參考風格：**
- 左側暖色（匹茲堡）漸變為右側冷色（台北）
- 監控元素用冷藍/冷綠穿插
- 類似 Orwell 遊戲的監控美學

**技術考量：**
- 以 `progress` (0–1) 控制元素浮現順序
- progress 0–0.3: 基礎場景
- progress 0.3–0.6: 電話線 + 郵件
- progress 0.6–1.0: 完整監控網顯現
- 手機降級：靜態完成圖

### 3. airport-return — 松山機場入境

| 項目 | 內容 |
|------|------|
| 章節 | 第三章：歸途 |
| 情緒氛圍 | 表面平靜但壓迫感遞增 |
| 互動方式 | 捲動顯示 |

**場景描述：**
1981 年松山機場入境大廳。陳文成提著行李通過海關。海關人員身後的暗處，隱約可見便衣人員的身影。出境許可被扣留的暗示——一張紅色戳記的文件。

**需要的元素：**
- 人物：陳文成（行李箱）、海關人員、暗處便衣 2–3 人
- 場景：機場入境通道、海關櫃檯、「入境」標示
- 關鍵道具：護照、被扣的出境許可（紅色戳記）
- 動畫：便衣人影若隱若現（alpha 脈動）

**參考風格：**
- 整體色調偏暗，機場日光燈的冷白光
- 便衣人員用剪影表示（不給細節，增加壓迫感）
- 色調從入口的明亮漸暗至通道深處

**技術考量：**
- 便衣人影用低 alpha sprite + 微動畫
- 靜態背景 + 陳文成行走動畫（2–3 幀）

### 4. interrogation-room — 警備總部審訊室

| 項目 | 內容 |
|------|------|
| 章節 | 第四章：約談 |
| 情緒氛圍 | 黑暗、壓迫、恐懼——全篇最暗色調 |
| 互動方式 | 捲動 + 對話框 |

**場景描述：**
狹小的審訊室。單一吊燈冷白光從上方照下，照亮桌面但四周陰暗。陳文成坐在桌子一側，對面是審訊者（可見輪廓但面容模糊）。桌上有文件、錄音機。

**需要的元素：**
- 人物：陳文成（疲憊、緊張）、審訊者 1–2 人（剪影/模糊面容）
- 場景：審訊室、金屬桌椅、吊燈
- 道具：文件堆、錄音機（暗示「約談有錄音但非全程」）、出境許可文件
- 光線：頂部吊燈的錐形冷白光，其餘全暗
- 對話框：捲動時顯示關鍵對話（「自己本身有安全問題」等）

**參考風格：**
- Papers, Please 審訊風格：極簡、壓迫
- 顏色幾乎只有灰黑 + 冷白燈光
- 人物細節極少，以剪影和姿態傳達情緒

**技術考量：**
- 燈光搖晃微動畫（1–2px 偏移循環）
- 對話框系統：與 ScrollSection progress 連動，逐條顯示
- 錄音機磁帶轉動微動畫

### 5. blank-hours — 空白的六小時

| 項目 | 內容 |
|------|------|
| 章節 | 第四章：約談（第三段） |
| 情緒氛圍 | 凝滯、未知——時間感停頓 |
| 互動方式 | 時鐘動畫 + 捲動推進 |

**場景描述：**
全黑畫面。畫面中央只有一個時鐘，指針從 21:30 走向翌日 07:00。隨捲動，時鐘緩慢前進，但畫面始終全黑——暗示這段時間的真相無人知曉。偶爾閃現碎片式的場景暗示（但模糊不清）。

**需要的元素：**
- 時鐘：圓形壁鐘，指針可動
- 時間標記：21:30 → 22:00 → ... → 07:00
- 閃現碎片（可選）：模糊的車輛、建築輪廓、人影——極低 alpha，一閃即逝
- 文字：隨捲動浮現——「無法證明警總確有將陳文成送返住處」

**參考風格：**
- 極簡主義——幾乎全黑
- 時鐘是唯一的視覺錨點
- 類似 Limbo 遊戲的黑暗美學

**技術考量：**
- progress 直接映射時鐘時間：0 = 21:30, 1 = 07:00
- 閃現碎片用極短 alpha fade in/out（200ms）
- 最低資源消耗的場景（幾乎無 sprite）
- 手機可保留——效能無壓力

### 6. library-dawn — 台大研究生圖書館旁

| 項目 | 內容 |
|------|------|
| 章節 | 第五章：陳屍台大 |
| 情緒氛圍 | 冷靜、沉重——以克制呈現 |
| 互動方式 | 捲動 + 緩慢揭露 |

**場景描述：**
清晨。台大研究生圖書館（五層白磚水泥建築）側面。太平梯可見。畫面下方有淺水溝。隨捲動，畫面從建築全景緩慢下移至地面——最後停在淺水溝旁，但不直接顯示遺體，以留白處理。

**需要的元素：**
- 建築：五層白磚水泥建築、消防太平梯、五樓護欄
- 環境：清晨薄霧、淺水溝、周圍植栽
- 人物（遠景）：校警巡邏的小小身影
- 光線：清晨灰藍色調，微光
- **不直接顯示遺體**——用留白、陰影或輪廓暗示

**參考風格：**
- 冷灰藍色調
- 建築物使用簡約幾何風格
- 霧氣效果增加沉重感
- 參考 Hyper Light Drifter 的環境敘事

**技術考量：**
- 鏡頭緩移效果：用 camera offset + progress 連動
- 霧氣：半透明 sprite 層，微動畫
- 護欄標示 83cm 高度（可加標註）

### 7. wecht-arrival — Wecht 抵台調查

| 項目 | 內容 |
|------|------|
| 章節 | 第六章：追尋真相 |
| 情緒氛圍 | 灰暗中偶有亮光——真相的希望 |
| 互動方式 | 捲動顯示 |

**場景描述：**
場景分三段（隨捲動切換）：
1. 殯儀館——Wecht 在檢驗桌旁檢視（冷色調）
2. 台大現場——Wecht 站在圖書館旁測量（灰色調）
3. 松山機場記者會——Wecht 在麥克風前（暖色調，「最可能的是謀殺」）

**需要的元素：**
- 人物：Wecht（西方面孔、法醫形象）、隨行人員、記者
- 場景 1：殯儀館內部、檢驗桌
- 場景 2：台大圖書館外（復用 library-dawn 背景）
- 場景 3：機場記者會、麥克風、閃光燈
- 暗處元素：國安局監控人員的身影（「全程監控」）

**參考風格：**
- 三段場景色調遞進：冷 → 灰 → 微暖
- Wecht 場景是全篇中「外來光明」的象徵
- 監控人員一以貫之地出現在暗處

**技術考量：**
- 三段場景用不同背景 sprite，progress 切換
- 可復用 library-dawn 的建築 sprite
- 監控人影持續出現（呼應 Ch2 surveillance-web）

## 通用技術規範

### Sprite 管理
- 所有場景 sprite 統一放在 `public/assets/sprites/` 下
- 每場景一個子資料夾：`sprites/campus-life/`、`sprites/surveillance-web/` ...
- 使用 sprite sheet 合併小元素，減少 HTTP 請求

### 效能原則
- 場景在進入 viewport 時才初始化（Intersection Observer）
- 離開 viewport 後釋放 PixiJS 資源
- 手機（< 640px）降級為靜態 PNG 截圖
- 目標：60fps 捲動，無掉幀

### 動畫連動
- 每個場景元件接收 `progress` prop (0–1)
- `progress` 由 `useScrollProgress` hook 提供
- 場景內部根據 progress 控制：元素顯示/隱藏、位移、alpha、動畫幀

### 無障礙
- 每個場景提供 `aria-label` 描述場景內容
- 靜態替代圖片用於螢幕閱讀器
- 動畫可由使用者偏好關閉（`prefers-reduced-motion`）
