# 資訊圖表規格

> 本文件列出所有建議的資訊圖表，供開發階段參考。
> 技術方案：SVG 或 CSS Grid/Flexbox，不使用 PixiJS（與 pixel art 場景區隔）。
> 所有圖表需支援 RWD，手機版提供降級方案。

## 1. surveillance-timeline — 監控時間線

| 項目 | 內容 |
|------|------|
| 章節 | 第二章：海外的聲音 |
| 圖表類型 | 橫向時間線 |
| 資料來源 | 促轉會調查報告 p.33–43；「彩虹資料」 |

**資料內容：**
- 五筆跨太平洋電話監聽紀錄（1979.09–1981.05）
  - 每筆含：日期、通話對象、監聽內容摘要
- 郵件攔截事件（時間點 + 攔截內容類型）
- 美麗島事件（1979.12.10）與林宅血案（1980.02.28）作為時代背景錨點

**視覺設計：**
- 橫向時間軸，左起 1979.09，右至 1981.05
- 監聽紀錄用節點標示，hover/點擊展開詳情
- 郵件攔截用不同圖示（信封 + 紅色攔截標記）
- 美麗島事件、林宅血案用醒目標記（`accent-red`）作為背景事件
- 色調：冷藍/冷綠（監控意象），背景暗色

**互動方式：**
- 桌面：hover 顯示 tooltip、點擊展開詳情卡
- 手機：點擊展開
- 捲動連動：進入 viewport 時，節點從左至右依序淡入

**RWD 降級：**
- 手機（< 640px）：改為垂直時間線，事件卡片全寬
- 平板（640–1024px）：保留橫向，但縮減 tooltip 內容

## 2. july-hourly-timeline — 核心 48 小時逐時時間線

| 項目 | 內容 |
|------|------|
| 章節 | 第三–四章（跨章） |
| 圖表類型 | 垂直逐時時間線 |
| 資料來源 | 促轉會調查報告 p.44–62, 64–87；Wecht Report p.3, 7–9 |

**資料內容：**
7月1日至7月3日，逐小時事件記錄：

| 時間 | 事件 | 重要性 |
|------|------|--------|
| 7/1 日間 | 追問出境證未果、航空公司有機位但無法出境 | turning-point |
| 7/1 晚間 | 白健二晚餐（1994 回述，可信度標註） | background |
| 7/2 05:00 | 台大操場打籃球 | background |
| 7/2 07:45 | 詢問妻子有無出入境管理局電話 | background |
| 7/2 08:30 | 三名男子持傳票帶走 | critical |
| 7/2 08:30–21:30 | 保安處約談（12小時） | critical |
| 7/2 ~21:30 | 警總稱約談結束、派車送返 | critical |
| 7/2 21:30–7/3 07:00 | **空白的六小時**（無可信證據） | critical |
| 7/3 清晨 | 校警接不知名電話 → 發現遺體 | critical |

**視覺設計：**
- 垂直時間軸，上至下對應時間推進
- 左右交替顯示事件卡片
- `critical` 事件用 `accent-red` 高亮邊框
- 空白的六小時用虛線 + 灰色區塊表示（「無可信證據」）
- 時間刻度明顯，每小時一格

**互動方式：**
- 捲動時高亮當前時間段（active state）
- 點擊事件卡片展開完整描述 + 來源標註
- 空白時段特殊處理：hover 顯示「現有證據無法證明...」

**RWD 降級：**
- 手機：單側顯示（全部靠左或靠右）
- 時間刻度簡化為關鍵時間點

## 3. testimony-comparison — 證詞矛盾對照表

| 項目 | 內容 |
|------|------|
| 章節 | 第四章：約談 |
| 圖表類型 | 並列對照表 |
| 資料來源 | 促轉會調查報告（下冊）p.64–87, 158–159 |

**資料內容：**
五名目擊證人 + 警總官方說法的交叉比對：

| 主題 | 警總說法 | 證人 A | 證人 B | 證人 C | ... |
|------|---------|--------|--------|--------|-----|
| 約談結束時間 | ~21:30 | （各說各話） | ... | ... | ... |
| 是否派車送返 | 是 | ... | ... | ... | ... |
| 陳文成離開狀態 | 心情很好 | ... | ... | ... | ... |
| 是否見到陳文成返家 | — | ... | ... | ... | ... |

特別標註：
- 鄧維祥證詞（促轉會質疑其可信度）
- 倪肇強證詞（亦不可信）
- 汪敬煦稱「心情很好」vs 筆錄的壓迫氣氛

**視覺設計：**
- 並列欄位，每欄一個證人/來源
- 矛盾之處用紅色標記 + 對角線連接
- 被質疑的證詞加註可信度標籤（低/存疑/不可信）
- 固定左欄（主題），橫向可捲動查看各證人

**互動方式：**
- 桌面：hover 高亮同一主題的所有欄位，便於比對
- 點擊矛盾標記展開詳細說明
- 手機：改為卡片式，每張卡片一個證人

**RWD 降級：**
- 手機：改為可切換的 tab（每個 tab 一個證人）
- 或改為垂直排列的摺疊面板

## 5. injury-diagram — 傷痕位置人體圖

| 項目 | 內容 |
|------|------|
| 章節 | 第五章：陳屍台大 |
| 圖表類型 | 人體標註圖（SVG） |
| 資料來源 | Wecht Report p.9–12；促轉會調查報告（下冊）p.92–106；陳文成之死因 |

**資料內容：**

體內傷：
- 右側第 2–8 肋骨骨折
- 脊椎骨折
- 右腎裂傷
- 其他內臟損傷

體外傷：
- 右背大面積擦傷
- **無**手部、腕部、指甲傷痕（= 無自我防禦傷）

關鍵推論：
- 傷勢集中右側 → 方向性
- 無防禦傷 → 失去意識或被控制
- 傷勢方向與陳屍位置不符 → 屍體可能被移動

**視覺設計：**
- 正面 + 背面人體輪廓（SVG，簡約線稿）
- 體內傷：半透明色塊覆蓋於對應位置（紅色系）
- 體外傷：表面標記（擦傷用橘色區塊）
- 「無傷」區域特別標註（手部、腕部）用綠色框 + 文字
- 標註線連接至側邊說明文字

**互動方式：**
- hover/點擊傷處展開法醫描述 + 來源頁碼
- 可切換：體內傷 / 體外傷 / 全部
- 進場動畫：傷處色塊逐一淡入

**RWD 降級：**
- 手機：保留 SVG，但說明文字改為下方列表
- 點擊傷處高亮並滾動至對應說明

| 項目 | 內容 |
|------|------|
| 章節 | 第五章：陳屍台大 |
| 圖表類型 | 圖文卡片列表 |
| 資料來源 | Wecht Report p.9–12；促轉會調查報告（下冊）p.159 |

**資料內容：**

| # | 疑點 | 說明 |
|---|------|------|
| 1 | 傷勢方向與陳屍位置不符 | 右側傷勢暗示右側著地，但陳屍姿態不符 → 屍體被移動 |
| 2 | 無自我防禦傷痕 | 手部、腕部、指甲均無傷 → 墜落時已失去意識 |
| 3 | 落地後仍存活半小時以上 | 體內出血量暗示存活時間 → 未獲救治 |
| 4 | 可能的致昏手法 | 頭部打擊或麻醉 → 非自主墜落 |
| 5 | 遺體如何進入校園？ | 深夜無車輛進出紀錄 → 校外運入可能性 |
| 6 | 五樓護欄事後重新上漆 | 毀滅可能的物理證據 |

**視覺設計：**
- 六張編號卡片，每張含：
  - 大編號（1–6）
  - 疑點標題
  - 簡圖/圖示（如：方向箭頭、手掌、時鐘、護欄等）
  - 展開後的詳細說明 + 來源
- 卡片排列：桌面 3×2 grid，手機 1×6 stack
- 色調：冷灰底 + `accent-red` 編號

**互動方式：**
- 點擊卡片展開詳細說明
- 捲動進場：卡片依序淡入（1→2→...→6）

## 通用技術規範

### 技術選型
- 靜態圖表：SVG（手繪或程式生成）
- 互動圖表：React 元件 + CSS 動畫
- 不使用 D3.js 等重量級圖表庫（控制 bundle size）
- 資料從 `docs/extracted/` 的 JSON 檔引入

### 無障礙
- 所有圖表提供 `aria-label` 和結構化文字替代
- 色彩對比度符合 WCAG AA
- 互動元素可用鍵盤操作（Tab + Enter）
- 不依賴顏色作為唯一區分手段（配合圖示/紋理）

### 效能
- SVG 圖表內嵌為 React 元件（非 img 引入）
- 複雜圖表使用 `React.lazy` + `Suspense`
- 動畫使用 CSS transform/opacity（觸發 GPU 加速）
- 避免強制重排（layout thrashing）

### 來源標註
- 每個圖表底部標示資料來源
- 使用 `<Citation>` 元件統一格式
- 格式：`（來源：促轉會調查報告，第 XX 頁）`
